记录下折腾和学习的过程

读<<黑客与画家>>

Nov 21, 2014 • admin

categories : 杂谈

扫描二维码

第一次听说Paul,是在学习sicp的时候,在程序设计技术和方法看到了阅读材料：一个新兴企业的故事.

于是搜索Paul后就发现了«黑客与画家». 但是当时有很多东西看过了没有办法理解,草草看过就放下了. 时间过了这么久了,至到之前无意间又看了王垠写的 «黑客文化的精髓»,说真的,突然感觉中了无数枪, 就像虽然«黑客与画家» 里面也写到说独立思考很重要, 但读完后难免不受到其影响 .

反思自己,未免不是在很多时候有随大流的做法,比如王垠提到的”择其难者而从之” ,当初开始学习、使用linux和Emacs的初衷未尝不是因为觉得很”酷”,很有”逼格”. 事实上第一次开始真正长时间使用Emacs就是因为学习sicp时使用了王垠的Emacs解决方案.

UNIX编程艺术 的作者EricS.Raymond 也是黑客精神的推崇者之一,他连同Paul Graham和许许多多黑客很多地方是相似的. 个人认为他们相似的地方中最值得一提的是,他们会不断反思自我,然后反思他们所崇尚的精神.

我,我们,许多人,黑客精神的崇拜者,对于黑客精神的理解也许还停留在”择其难者而从之”的阶段,所以,在双十一我买了这本书的实体版,好好再看一遍.

ps : 对于 UNIX编程艺术 ,我认为我还缺乏大量实际开发经验,现在很多东西无法真正理解,等我有了更多经验,再将这本书加入我的收藏 : ) (之前看的是在图书馆借的,马上到期了,对于借到书以后发现 我是我们学校图书馆第一个借阅这本书的人,我感到很诧异)
